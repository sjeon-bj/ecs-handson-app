以下の手順でGitコミットを作成してください：

1. `git status`を実行して、変更されたファイルと未追跡ファイルを確認
2. `git diff`を実行して、ステージングされていない変更内容を確認
3. `git diff --staged`を実行して、ステージング済みの変更内容を確認
4. `git log --oneline -5`を実行して、最近のコミットメッセージのスタイルを確認

上記の情報を基に：
- 変更内容を分析し、適切なコミットメッセージを日本語で作成
- 必要なファイルをステージングエリアに追加（`git add`）
- コミットを実行（コミットメッセージは日本語で、変更の「why」に焦点を当てる）
- コミット後に`git status`で成功を確認

注意事項：
- シークレット情報を含む可能性のあるファイル（.env、credentials.jsonなど）はコミットしない
- コミットメッセージは簡潔に1-2文で作成
- Heredocを使用してコミットメッセージを渡すこと

例：
```bash
git commit -m "$(cat <<'EOF'
ユーザー登録機能を追加

新規ユーザーがアカウントを作成できるようにするため、
フォーム検証とパスワードハッシュ化機能を実装
EOF
)"
```
